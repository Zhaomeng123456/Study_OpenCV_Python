# OpenCV 28 天学习计划 — 开发准则

本文档是本项目每日练习代码的统一开发规范。新增 Day2 ~ Day28 内容时，请严格遵循本准则。

---

## 一、项目定位

- 学习来源：[`OpenCV_28天学习计划表_跟随AI编程.xlsx`](OpenCV_28天学习计划表_跟随AI编程.xlsx)
- 组织方式：按天建目录 `Day1/` ~ `Day28/`，每天实现 Excel 中对应的实操任务
- 代码定位：学习练习脚本，追求**简洁可读**，而非工业级工程封装

## 二、目录结构规范

每一天的学习内容放在独立目录中，结构如下：

```text
DayN/
├── images/          # 测试图片（用户自行放入）
├── output/          # 脚本输出结果
└── dayN_xxx.py      # 当日练习脚本（可多个）
```

**路径约定：**

- 测试图片：放在 `DayN/images/`
- 输出文件：保存到 `DayN/output/`
- 代码中统一使用 `os.path.join(os.path.dirname(__file__), ...)` 拼接路径，避免硬编码绝对路径

## 三、代码风格规范

遵循 **Python 标准风格（PEP 8）**，同时保持**简洁**，遵循以下原则：

| 要求 | 说明 |
|------|------|
| 简洁直线式 | 从上到下顺序执行，逻辑清晰，避免不必要的抽象 |
| 单文件操作为主 | 未明确要求时，只处理一张图片，不做文件夹遍历/批量处理 |
| 适度封装 | 简单脚本可不写 `main()`；逻辑变复杂时再提取函数 |
| 精简错误处理 | 不做冗长的异常提示，保持学习脚本简洁 |
| 命名规范 | 变量、函数使用 snake_case；常量使用大写 |
| 导入顺序 | 标准库 → 第三方库（`import cv2`）→ 空一行再写代码 |
| 窗口显示 | 按实际需求选用 API，不强制统一方案（见下文） |

**窗口显示：按需选用**

| API | 适用场景 |
|-----|----------|
| 直接 `cv2.imshow()` | 快速预览，窗口大小随图像自动适配 |
| `cv2.namedWindow()` | 需要预设窗口属性（如可拖拽缩放）时使用 |
| `cv2.resizeWindow()` | 需要在显示前指定窗口初始尺寸时使用 |

示例（需要自定义窗口时）：

```python
window_name = "preview"
cv2.namedWindow(window_name, cv2.WINDOW_NORMAL)
cv2.resizeWindow(window_name, 800, 600)
cv2.imshow(window_name, img)
```

示例（简单预览时）：

```python
cv2.imshow("preview", img)
```

## 四、注释规范

遵循 Python 标准注释惯例（PEP 8 / PEP 257），**按用途区分**，不统一只用 `#`。

| 注释类型 | 写法 | 用途 |
|----------|------|------|
| 模块文档字符串 | 文件顶部 `"""..."""` | 项目介绍、本阶段新学/不常见算子说明 |
| 行内注释 | `# 说明` | 代码中间的关键步骤说明 |
| 函数文档字符串 | 函数内 `"""..."""` | 仅在定义函数时，说明函数用途与参数 |

### 4.1 模块文档字符串（文件头部）

每个 `.py` 文件顶部使用**三引号 docstring**，包含：

**（1）【项目介绍】**

- 本文件属于 OpenCV 28 天学习计划的第几天
- 当天的学习主题与学习目标

**（2）【重点算子】**

仅介绍**本阶段/当天新学的算子**和**不常见的算子**，已在前期学过的常用算子**不再重复**写在头部。

**需要写入头部的算子：**

- 当天 Excel 学习计划中列出的核心知识点对应 API
- 本文件首次使用、容易混淆或参数较多的算子
- NumPy 中首次用于图像处理的操作（如切片、`.copy()`、`np.mean()`）

**不需要写入头部的常用算子：**

以下算子在前期已反复使用，代码中直接调用即可，必要时用行内 `#` 注释一句：

| 常用算子 | 说明 |
|----------|------|
| `cv2.imread()` | 读取图片 |
| `cv2.imwrite()` | 保存图片 |
| `cv2.imshow()` | 显示图像 |
| `cv2.waitKey()` | 等待按键 |
| `cv2.destroyAllWindows()` | 关闭窗口 |
| `cv2.VideoCapture()` | 打开摄像头/视频 |
| `cap.read()` / `cap.release()` | 读帧 / 释放摄像头 |
| `os.path.join()` | 路径拼接 |

写入头部的每个算子，须包含：

| 要素 | 说明 |
|------|------|
| 功能 | 这个算子做什么 |
| 参数 | 列出全部参数 |
| 参数含义 | 每个参数代表什么 |
| 可设范围 / 常用取值 | 如枚举值、数值范围、常用选项 |
| 返回值 | 返回什么类型、什么含义 |

**（3）通用避坑提醒**（如适用）

示例（Day2 只介绍当天新学的 ROI / 像素操作，省略 imread 等常用算子）：

```python
"""
OpenCV 28天学习计划 — Day2：像素操作与ROI区域

【项目介绍】
本项目是 OpenCV 28 天系统学习计划，按天练习工业视觉常用技能。
Day2 学习目标：掌握图像矩阵属性、像素读写、ROI 区域裁剪。

【重点算子】

1. img.shape
   功能：获取图像矩阵的尺寸信息
   参数：无
   返回值：(高度, 宽度) 或 (高度, 宽度, 通道数)

2. img[y, x]
   功能：读取或修改单个像素值
   参数：
     - y (int)：行索引，范围 0 ~ 高度-1
     - x (int)：列索引，范围 0 ~ 宽度-1
   返回值：像素 BGR 值

3. img[y1:y2, x1:x2]
   功能：通过 numpy 切片裁剪 ROI 区域
   参数：
     - y1:y2 (int)：行范围（左闭右开）
     - x1:x2 (int)：列范围（左闭右开）
   返回值：ROI 视图，修改会同步影响原图

4. roi[:] = (B, G, R)
   功能：批量修改 ROI 内所有像素颜色
   参数：(B, G, R) 元组，各分量范围 0~255
   返回值：无

注意：OpenCV 默认使用 BGR 通道顺序，与 matplotlib/PIL 的 RGB 不同。
"""

import os

import cv2
```

### 4.2 行内注释（代码中间）

在关键步骤上方或行尾用 `#` 添加简短中文说明：

```python
# 读取本地图片
img = cv2.imread(path_img)

# 裁剪 ROI 区域（行 50~200，列 100~300）
roi = img[50:200, 100:300]

# 将 ROI 区域像素改为红色（BGR: 蓝0, 绿0, 红255）
roi[:] = (0, 0, 255)
```

## 五、脚本拆分原则

按当天实操任务的独立性决定是否拆分文件：

| 场景 | 做法 |
|------|------|
| 当天只有一个连贯任务 | 一个脚本完成 |
| 当天有多个独立任务 | 拆成多个脚本（参考 Day1：图片读写 + 摄像头预览） |

**命名规范：** `dayN_功能描述.py`，例如：

- `day1_image_io.py`
- `day1_camera_preview.py`
- `day2_roi_crop.py`
